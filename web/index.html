Welcome to Network Coding in WiMAX Demo

Our demonstration showcases a network-coding (NC)-enabled reliability architecture for next generation wireless networks. Our NC architecture uses a flexible thread-based design, applying systematic intra-session random linear network coding as a packet erasure code at the IP layer. Using GENI WiMAX platforms, a series of point-to-point transmission experiments are conducted to compare NC with Automatic Repeated reQuest (ARQ) and Hybrid ARQ (HARQ). At the application layer, Iperf and UFTP are used to measure throughput, packet loss and file transfer delay. In our selected scenarios, NC offers up to 5.9 times gain in throughput and 5.5 times reduction in file transfer delay, compared to HARQ and joint HARQ/ARQ. Our demonstration hence illustrates that lower-layer redundancy mechanisms such as HARQ and ARQ incur high cost since they operate at the packet-level. Conversely, running at higher layers (e.g., IP) amortizes the cost of redundancy over several packets, thus leading to higher efficiency.

Experimental Setup

  • Four fixed uplink MCS/tx. power settings
  • Default MAC/PHY parameters used in BS and SS (listed below)
  • Throughput/Loss Experiment: Fixed offered load (6Mbps) and duration (30s)
  • File Transfer Experiment: Fixed offered load (6Mbps) and file size (10MB)
  • Reliability Configurations:
    • Raw: No applied reliability scheme
    • HARQ (CC)
    • Joint HARQ-ARQ
    • NC configurations denoted NC-m, where m is the redundancy parameter (i.e., percentage of redundancy packets per block)
  • Simple demonstrated configuration:
    • Single round of redundancy packets per block, sent a priori immediately after systematic (uncoded) packets
    • Block size = 120 packets
    • Coded packet size = 1400 bytes
    • Redundancy factor from 0-100%
